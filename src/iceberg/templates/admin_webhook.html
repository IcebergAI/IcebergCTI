Publication webhook

When a report is published, Iceberg POSTs a small JSON event (report id, title, link and recipient count — metadata only, never the report body) to this endpoint as a background task. Delivery is failure-isolated and routed through the global outbound proxy when one is configured. The bearer token is read from the environment and never stored here.
